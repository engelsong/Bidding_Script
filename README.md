# Bidding Script

用于援外项目投标文件的自动化生成。项目目前已经提供统一的控制台入口，用户可以在 `main.py` 中按需选择生成报价表、目录、封面、空白文件夹结构，或拆分报价表。

## 项目概览

当前项目围绕一个核心输入文件工作：

- `project*.docx`

程序会从该 Word 文件中提取项目名称、招标编号、开标日期、物资清单、服务需求等信息，并生成对应的投标辅助文件。

仓库中保留了一个项目 Word 模板：

- `project-[项目名称务].docx`

当前所有主要功能都已经接入 `main.py`，可以通过控制台菜单直接调用，不需要再手动修改入口代码。

## 当前功能

- 读取并解析 `project*.docx`
- 生成报价表 Excel 文件
- 生成目录 Excel 文件
- 生成封面 Word 文件
- 生成空白投标文件夹结构
- 将报价表按指定工作表拆分为多个单独的 Excel 文件

## 模块说明

- `main.py`
  统一入口脚本，负责显示菜单、接收用户输入、按顺序调用对应功能，并处理覆盖确认和异常提示。

- `project.py`
  负责读取 `project*.docx`，解析项目基础信息、物资清单、需求信息、法检物资、主要标的等，供其他模块使用。

- `quotation.py`
  负责生成完整报价表，是当前项目的核心业务模块。输出文件为：
  `投标报价表-<项目名>.xlsx`

- `content.py`
  负责生成投标目录文件。输出文件为：
  `目录-<项目名>.xlsx`

- `cover.py`
  负责生成投标封面文件。输出文件为：
  `封面-<项目名>.docx`

- `directory.py`
  负责根据项目内容创建空白投标文件夹结构。输出结果为：
  `投标文件-<项目名>` 目录及其子目录

- `separate.py`
  负责读取已有报价表，将指定工作表拆分成单独文件，并尽量把公式结果固化为数值。输出结果为多个独立的 Excel 文件，例如：
  `1.投标报价总表.xlsx`

## 依赖环境

- Python 3.10+
- `python-docx`
- `openpyxl`

安装依赖：

```bash
pip install python-docx openpyxl
```

如果希望 `separate.py` 在拆分时尽量把公式结果稳定固化为数值，建议本机安装 Microsoft Excel。没有 Excel 时，程序仍可拆分报价表，但结果依赖原工作簿中已有的公式缓存值。

## 输入文件

将项目 Word 文件放在项目根目录，文件名需匹配：

- `project*.docx`

示例：

- `project-[项目名称务].docx`

注意：

- 建议同目录只保留一个匹配的 Word 文件
- 当前入口会使用当前目录中最后匹配到的 `project*.docx`
- 模板表格结构和字段顺序会影响解析结果，新增字段前建议先同步调整 `project.py`

## 运行方式

运行入口脚本：

```bash
python main.py
```

启动后，控制台会显示功能菜单：

```text
请按照序号选择你需要的功能：
1、生成报价表
2、生成目录
3、生成封面
4、生成空白本文件夹结构
5、拆分报价表
>>> 
```

输入规则：

- 可输入单个编号，例如 `1`
- 可连续输入多个编号，例如 `125`
- 多个功能会按输入顺序执行
- 重复编号会自动去重
- 输入为空或包含非法字符时，程序会提示重新输入

功能说明：

- `1`
  生成报价表 Excel 文件

- `2`
  生成目录 Excel 文件

- `3`
  生成封面 Word 文件

- `4`
  生成空白投标文件夹结构

- `5`
  拆分已有报价表，将符合命名规则的工作表导出为单独 Excel 文件

执行逻辑说明：

- 选择 `1`、`2`、`3`、`4` 时，程序会先读取 `project*.docx`
- 仅选择 `5` 时，不依赖 `project*.docx`，会直接查找已有 `投标报价表-*.xlsx`
- 当目录中存在多个 `投标报价表-*.xlsx` 时，拆分功能会使用最近修改的报价表
- 当目标文件已经存在时，程序会提示是否覆盖

## 输出结果

根据所选功能，程序会在项目根目录生成以下文件或目录：

- 报价表：
  `投标报价表-<项目名>.xlsx`

- 目录：
  `目录-<项目名>.xlsx`

- 封面：
  `封面-<项目名>.docx`

- 文件夹结构：
  `投标文件-<项目名>/...`

- 拆分后的报价表：
  `1.投标报价总表.xlsx`
  `2.采购需求偏离表(物资部分).xlsx`
  `3.开标一览表.xlsx`
  以及其他符合拆分规则的工作表文件

## 报价表说明

`quotation.py` 是当前项目中最核心的业务模块，负责根据 `Project` 解析出的项目数据直接生成完整报价表，不依赖 Excel 模板。

主要能力包括：

- 创建完整报价表工作簿
- 生成多个业务工作表
- 根据物资清单、服务需求、培训需求等数据填充内容
- 写入金额汇总、费用分摊、税额计算等公式
- 输出最终报价表 Excel 文件

当前报价表通常包含以下工作表或同类工作表：

- 全部厂家备用
- 物资选择
- 投标报价总表
- 开标一览表
- 采购需求偏离表
- 对内分项报价表
- 各项物资退抵税额表
- 技术服务费报价表
- 来华培训费报价表
- 三体系一览表

实现特点：

- 报价表由代码直接生成，不依赖模板
- 工作簿中大量金额和汇总单元格使用公式
- 已启用 `wb.calculation.fullCalcOnLoad = True`
- Excel 打开文件后会触发重算
- 新版 Excel 可能会将部分公式显示为 `=@INDEX(...)`，这是动态数组兼容行为，通常不影响计算结果

## 拆分功能说明

`separate.py` 的主要逻辑如下：

- 查找 `投标报价表-*.xlsx`；如果存在多个匹配文件，使用最近修改的文件
- 尝试调用本机 Excel 打开并保存一次，以刷新公式缓存值
- 使用 `data_only=True` 重新读取工作簿
- 将工作表名匹配 `^[0-9]{1,2}\.` 的工作表分别导出为单独文件

拆分后的单独文件适合直接提交、打印或单表检查。

## 目录与封面说明

`content.py` 生成的是目录 Excel 文件，主要用于投标文件目录整理和页码填写。

`cover.py` 生成的是封面 Word 文件，适合后续人工微调排版或直接导出打印。

`directory.py` 生成的是空白目录结构，便于将商务标、技术标、经济标等资料按固定结构归档。

## 注意事项

- `openpyxl` 不会计算 Excel 公式，只能读取 Excel 已保存的缓存结果
- 如果未安装 Excel，拆分后公式单元格是否能稳定转换为数值，取决于源报价表是否已经被 Excel 打开并保存过
- 当前项目中存在一些历史代码和注释，使用方式以 `main.py` 的菜单入口为准
- 如果同目录存在多个 `project*.docx`，建议先清理，只保留当前要处理的项目文件
- `__pycache__/`、`*.pyc` 和 `.idea/` 已通过 `.gitignore` 忽略，不再作为项目源码提交
- `project-[项目名称务].docx` 是项目模板，应继续保留在版本库中
